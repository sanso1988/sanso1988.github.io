Chapter 7 Regularization for Deep Learning

The key problem in marchine learning is to make an algorithm perform well on new inputs, we call this property generalization. Regularization is for improving the algorithm's generalization, the definition of regularization in this book is "Any modification we make to a learning algorithm that is intended to reduce its generalization error but not its training error".

There are many strategies used in regularization. Some impose extra constraints on parameters, some add extra terms in the objective function. These constraints or penalties sometimes reflect the prior knowledge we believe we have, sometimes just wanna keep model simpler to promote generalization.

In the context of deep learning, most regularization strategies are based on regularizing estimators. It works by trading bias for reduced variance. An effective regularizer is one that reducing the variance significantly but not overly increasing the bias.

In practice, almost any overly complex model family couldn't actually includes data-generating process in the real world. Instead, we might find that the best fitting model (in the sense of minimizing generalization error) is a large model that has been regularized appropriately.

Parameter Norm Penalties

Regularization was introduced in decades ago prior to the advert of deep learning for the linear model. The main approach is to add parameters norm penalty term to limit the capacity of models.

J~(θ;X,y)=J(θ;X,y)+αΩ(θ)

where α∈[0,∞) is a hyperparameter that weights the relative contribution of the norm penalty term, the larger one corresponds to more regularization.

Different choices of norm function leads to different solutions being preferred. The book discusses the effects of various norms.

In deep learning, it's normal to use norm penalty only on the affine parameters, level the biases unregularized. The book illustrates this as biases don't induce too much variance but crucial for fitting appropriately. Therefore, we denote the regularized parameters as ω and all the parameters as θ. In my persperctive, biases are key factor to diminish the error caused by features designed arbitrarily.

In the context of neural networks, it's desirable use a different α coefficient on different layers. Due to the expense to search for the correct value of all coefficients, it's still resonable to use the same weight decay at all layers.

L2 Parameter Regularization

L2 norm is most common penalty in regularization, the objective funtion added by Ω(θ)=12||w||22. And it's far more common to regularize the parameters towards zero while we don't have specific knowledge about what's ture value of the parameters.

The book demonstrates the effect of L2 regularization by assuming the loss function as quadratic. The regularized objective function is as below.

J~(θ;X,y)=J(θ;X,y)+12wTw

Modified objective function leads to a new gradient which has a shrinking effect on parameters update. As using Taylor expansion formula, we could find out that how's the new minimum point different from the original one.

w~=(H+αI)−1Hw∗

Cauze H is symmetric, it can be decomposed into a diagonal eigenvalues matrix Λ and an orthonormal eigenvectors matrix Q, such that H=QΛQT.

w~=Q(Λ+αI)−1ΛQTw∗

Obviously, the effect of weight decay (L2 Regularization) is to rescale the original minimum point w∗ along the eigenvectors of Hessian matrix, the rescaled factor is λiλi+α.

If λi is far smaller than α which means increasing distance between w∗ along this direction leads to slightly increasing loss of J, the component is shrunk to nearly zero. If λi is far larger than α, the component is almost not changed.

As assuming quadratic objective funtion, the regularized minimum point is actually the balance point between the original point and zero point according to the sensitivity between loss function and regularized function.

L1 Parameter Regularization

L2 is a common norm used with preferable mathmatical property, Other norm is also available for weight decay, L1 norm defined as below is the option of regularization too.

Ω(θ)=||w||1=∑|wi|

The regularized objective function truns to be as below.

J~(w)=J(w)+αΩ(θ)

The effect of this kinda regularization is hardly analyzed mathematically, new minimum point is sparse, which means lots of weights are zero. The sparsity property induced by L1 regularization has been used as a feature selection mechanism, it could sift out subset of useful features.

Norm Penalties as Constrained Optimization

Instead of adding penalty in objective function, we also can make some hard constraints, like that require Ω(θ)<k. As dicussed before, this problem could be solved by constructing a Lagrange function including KKT multiplier α.

L(θ)=J(θ)+α(Ω(θ)−k)

θ∗=argminθmaxα>=0L(θ)

θ is normal to be solved via gradient descent. it means increasing α at the point where Ω(θ)>k, decrease α at others.

Sometimes, we may wish to use explicit constraints rather than Lagrange penalties, it takes the down hill step at every valid point which satisfies the constraints. For this, one reason is that we know k exactly, we don't want any violations. Second reason is that penalty regularization objective funtion could get stuck in local minima corresponding to small parameters, even increasing the parameters could improve the performance significantly. Third reason is that it could impose some stability on the optimization process, we can use a high learning rates without worrying numerical overflow too much of constraints pull it back.

Regularization and Under-Constrained Problems

In this section, book discusses about the relationship between regularization and underdetermined problems. The core idea is that regularization is useful for solving underdetermined problems.

For example, linear regression or PCA, depend on inverting the matrix XTX. If the matrix is singular, there is no exact solution. Actually, XTX is the covariance matrix of observations. In this case, many forms of regularization correspond to inverting XTX+αI instead.

Not just only linear models face this kind of problem but nonlinear models too. An example is logistic regression applied to a problem where the classes are linearly separable. If w is able to achieve perfect classification, 2w will also achieve perfect classification and higher log likelihood, iterative optimization process are not going to terminate.

Most forms of regularization are able to guarantee the convergence of iterative optimization process. The idea of using regularization to solve underdetermined problems extends far beyond machine learning.

Dataset Augmentation

The best way to improve generalization is to increase training data, but it regularly imposes more expense and sometimes it's unrealistic. For some tasks, it's resonably straightforward to create new fake data to generalize the model.

This approach is very useful for some classification problems, especially image recognition, cause the category should not change to a wide variety of transformations. Operations like translating the training images a few pixels in each direction can often greatly improve generalization.

Injecting noise can also be seen as a form of dataset augmentation. Neural networks prove not to be robust to noise, one way to improve the bobustness is to train them with random noise applied to inputs. Noise injection also works when the noise is applied to the hidden units. Dropout, a powerful regularization strategy is also seen as a form of noise injection.

Due to the significant effect, taking dataset augmentation into account is important while comparing different machine learning benchmark performance.

Noise Robustness

In the general case, it's important to remember that noise injection can be much more powerful than shrinking the parameters, especially when the noise is added to hidden units.

In addition to injecting noise to inputs, noise also could be added to the weights, which could be interpreted as a Bayesian inference. This regularization strategy encourage stability of the function to be learned.

As using MSE loss, this strategy is equivalent to adding regularization term of predictor gradient norm to objective function. It pushes the parameters into a region which is not merely minima but flat.

Injecting Noise at the Output Targets

Adding noise to the label is a solution for the situation that there are some number of mistakes in the labels. Label smoothing replacing the hard 0 and 1 classification targets with targets of ϵk−1 and 1−ϵ, respectively. It has the ability to prevent the unrelenting pursuit of hard probabilities which may never converge.

Semi-Supervised Learning

In the context of deep learning, semi-supervised learning refers to learning a representation h=f(x) which is helpful to handle the task. Examples that cluster tightly in the input space should be mapped to similar representation, a linear classifier in the new space may achieve a better generalization.

Long-standing variant of this approach is the application of PCA as a preprocessing step before applying a classifier.

Multitask Learning

Multitask learning is a way to improve generalization via sharing part of model within multiple tasks, which can be seen as soft constraints imposed on the parameters.

Achievement could be gotten because of the shared parameters are trained with more examples. Of course this will happen only if shared parameters assumptions are set appropriately.

The belief behind the idea is that among the factors that explain the variations observed in the data associated with the different tasks, some are shared across two or more tasks.

Early Stopping

Early stopping refers to that we stop the training process early, especially at the point we obtain a model with better validation error. When training large models with large capacity to overfit the task, it's common that training error decreases steadily over time, but validation error follows a U-shape curve.

Early stopping is probably the most commonly used form of regularization strategy in deep learning, due to effectiveness and simplicity.

Early stopping can be seen as a hyperparameter selection strategy by considering stop steps as hyperparameter.

The cost to implement early stopping strategy is small, in terms of validation error computation time and best parameters store memory.

Early stopping requires a extra validation set, which reduces the number of examples for training. There are two ways to incorporate the validation set into training, one is to retrain the model on all examples from the start point and stop it at the pre-stop steps, the other is to train the model on all examples from previous best point and stop it until the training loss of validation set falls below the training error on previous training set of previous best point.

Under certain assumptions like quadratic loss function, early stopping strategy behaves almost as same as L2 regularization. Actually, early stopping is far beyond L2 regularization, it involves monitoring validation set error in order to stop training, not need weight decay hyperparameter at all.

Parameter Tying and Parameter Sharing

In addition to adding some penalties to the objective function, we could use another way to express our prior knowledge of the parameter domain. Sharing Parameters refers to force sets of parameters to be equal.

Sharing parameters reduces the memory needed to store the parameters, it enables model to significantly increase network sizes without requiring a corresponding increase in training data. The quintessential example of parameters sharing is CNN.

Sparse Representations

Sparse representations refers to impose a penalty on the activations of the units in a neural network instead of on the model parameters, it encourages the representations to be sparse (many to be zero or close to zero).

Essentially any model that has hidden units can be made sparse. It likes transforming the inputs and extracting the important ones.

Bagging and Other Ensemble Methods

Bagging shorts for bootstrap aggregating, is a technique for reducing generalization error by combining several models vote on the output of examples (ensemble methods). As assuming models produce a zero-mean with identical variance errors, obviously ensemble could gain a better estimator.

Different ensemble methods construct the ensemble of models in different ways. Bagging constructs k different datasets (resample with replacement from original dataset), allows the same kind of model, training algorithm and objective function to train on k datasets, then vote to generate the ensemble.

Model averaging is an extremely powerful and reliable method for reducing generalization error.

Dropout

Dropout provides a computationally inexpensive but powerful method of regularizing a broad family of models. Dropout can be seen as a ensemble methods which is making bagging practical for large neural networks.

Dropout trains the ensemble consisting of all subnetworks that can be formed by removing nonouput units from a base network. For main situation, it generates a μ comprising of every switch for units to multiply. The Dropout objective function is EμJ(θ,μ).

In practice, we commonly use a mini batch-based learning Algorithm that makes small steps, such as stochastic gradient descent. Each time we load a example into a minibatch, we randomly produce a binary mask to each qualified unit, then run forward propagation, back-propagation and the learning update as usual.

To make a prediction, we could use the arithmetic mean over the distribution of μ like ∑μp(μ)p(y|x,μ). It's almost impossible to calculate this in large network model. Mote Carlo is commonly practical approach to get the estimating value. A better approach is to use geometric mean over the distribution via weight scaling inferece rule. The latter approach is not theoretically proved but often works well in nonlinear model. Generally, the optimal choice of inference approximation is problem dependent.

Dropout is more effective than other standard regularization strategies, such as weight decay, sparse, etc. It's also very computationally cheap and applicable unanimously.

When extremely few labeled training examples are available, dropout is less effective. When the number of examples are large enough, dropout is unnecessary.

The stochasticity is not necessary or sufficient to achieve the regularizing effect of dropout.

Dropout has inspired other stochastic approaches to training exponentially large ensembles of models that share weights like DropConnect, Stochastic polling, etc.

One of the key insights of dropout is that training a network with stochastic behavior and making predictions by averaging over multiple stochastic decisions implements a form of bagging with parameter sharing. In principle, any kind of random modification is admissible. For the sake of prediction convenience, ideally we should use μ of unit-mean.

It's important to note the powerful generalization of dropout that comes from the multiplicative noise applying to hidden units. The book illustrates this idea in paradigm of image recognition

Adversarial Training

In many cases, even neural networks that perform at human level accuracy have a nearly 100 percent error rate on examples that are intentionally constructed, what we call it adversarial examples.

Adversarial training refers to that we construct adversarial examples to train original neural networks. One can reduce the error rate on the test set via adversarial training.

Adversarial training discourages highly sensitive locally linear behavior by encouraging the network to be locally constant in the nerborhood of the training data. This can be seen as a way of explicitly introducing a local constancy prior into supervised neural nets.

Tangent Distance, Tangent Prop and Manifold Tangent Classifier

Tangent distance of points refers to the distance between the manifolds corresponding to each point. A cheap alternative that makes sense locally is to approximate the manifold by the point tangent, therefore calculate the distance between tangents.

Tangent prop algorithm trains a neural net classifier with an extra penalty to make each output f(x) of the neural net locally invariant to known factors of variation. Local invariance is achieved by requiring Δxf(x) to be orthogonal to the known manifold tangent vectors v(i) at x.

Ω(f)=∑(Δxf(x)Tv(i))2

where the tangent vectors are derived a priori, usually from the formal knowledge of the effect of transformation, such as translation, rotation and scaling.

Compare to dataset augmentation, it has two major drawbacks. First, it only regularizes the model to resist infinitesimal perturbation. Second, it poses difficulties for models based on rectified linear units which can only shrink their derivatives by turning off or their weights, but not by saturating at a high value with large weights.

The manifold tangent classifier eliminates the need to know the tangent vectors a priori, auto encoders can estimate the manifold tangent vectors.
