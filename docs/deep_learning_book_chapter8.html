Chapter 8 Optimization for Training Deep Models

Deep learning algorithms involve optimization in many contexts, the most difficult one is neural network training. This chapter presents these optimization techniques for neural network training.

This chapter focuses on that finding the parameters θ of a neural network that significantly reduce a cost function J(θ), which typically includes a performance measure over the entire training examples as well as additional regularization terms.

The contents are arranged in terms of Differs, Challenges, Algorithms and Combinations.

How Learning Differs from Pure Optimization

The most important difference from traditional optimization algorithms is indirect. In most machine learning scenarios, we care about some performance measure on the test set and may be intractable. We reduce a different cost function J(θ) in the hope that doing so will improve the performance measure on the test set. In contrast to this, pure optimization algorithms minimizing J(θ) is a goal and of itself.

Typically, the cost function can be written over training set as below.

J(θ)=E(x,y)∼p^dataL(f(x;θ),y)

We usually prefer to optimize the objective funtion on real data generation distribution as below.

J∗(θ)=E(x,y)∼pdataL(f(x;θ),y)

Empirical Risk Minimization

The goal of machine learning is to reduce the generalization error which is on the base of true data generation distribution. If we know the real distribution, the training problem convert into the traditional optimization problem.

In most cases, we don't know the true distribution but get some examples or observations, the simplest way to tackle with this is that minimize the expected loss on the training set. We call it empirical risk as below.

Ex,y∼p^data(x,y)[L(f(x,θ),y)]=1m∑L(f(xi,θ),yi)

where m is the number of observations.

There are two problems as directly using empirical risk minimization. One is that empirical risk minimization is prone to overfitting. The other is that some loss function, such as 0-1 loss, don't have useful gradient which is not feasible for optimization.

Alternatively, we must use a slightly different approach, in which the quantity that we actually optimize is even more different from the quantity we truly want to optimize.

Surrogate Loss Function and Early Stopping

For some loss function we care about can not be optimized efficiently, we need to use surrogate loss function instead. For example, the negative likelihood is typically used as a surrogate for 0-1 loss function.

In some cases, surrogate loss function could learn more. For example, negative likelihood function could be continued to decrease as the best classification result reached.

Early stopping makes the optimization of machine learning not usually halt at a local minimum.

Batch and Minibatch Algorithms

One aspect of machine learning algorithms is that the objective function usually decompose as a sum over training examples. The gradient can be seen as the expectation, also has the linear property over the training set.

ΔθJ(θ)=E(x,y)∼p^dataΔθL(f(x;θ),y)

Computing the exact expectation or estimation is very expensive. In practice, we randomly select some examples from the training set to make the estimation.

The standard error of n examples is σn, which decreases much slower than the cost of computation increases as n get larger. Nonetheless, we may not use too small n, cause the standard error is too big which means the estimated gradient may be far beyond the ture value which is harmful for optimization.

Minibatch sizes are generally driven by the following factors:

  • Larger batches provide a more accurate estimator but with less than linear returns.
  • Multicore architectures are usually underutilized by extremely small batches.
  • If all examples in the batch are to be processed in parallel, then the amount of memory scales scales with batch sizes, which is the limitation for many hardware setup in batches.
  • Some kinds of hardware achieve better runtime with specific sizes of arrays. Especially GPUs need the number to be the power of 2 from 32 to 256.
  • Small batches can offer a regularizing effect, perhaps due to the noise added to the learning process.

Different optimization algorithms may prefer different batches, some algorithms are more sensitive to estimation error than others. For example, the optimization only using gradient can handle small batches like 100, the optimization with second-order methods typically require much larger batches like 10000.

It's crucial that the minibatches be selected randomly. An unbiased estimated gradient requires the observations to be independent. In practice, it's usually sufficent to shuffle the order of the data set once. Failing to shuffle the examples in any way can seriously reduce the effectiveness of algorithms.

Many optimization problems in machine learning decompose over examples well enough that we can compute entire separate updates over different examples in parallel. we call this asynchronous parallel distributed approaches.

In online learning, examples or minibatch are drawn from a stream of data, no repeated data is used. The estimator of gradient is always unbiased, stochastic gradient descent performs on the generalization error.

Nevertheless, it is usually best to make several passes through the training set, unless the training set is extremely large. when multiple such epochs are used, only the first epoch follows the unbiased gradient of the generalization error, but the additional epochs usually provide enough benefit due to decrease the training error to offset the harm they cause by increasing the gap between training error and test error.

With some datasets growing rapidly in size, faster than computing power, it become common for machine learning applications to use each training example only once or even to make an incomplete pass through the training set. when using extremely large training dataset, overfitting is not an issue, underfitting and computational efficiency become the predominant concerns.

Challenges in Neural Network Optimization

Traditionally, machine learning try best to avoid the difficulty of general optimization by carefully designing the objective function. When training neural networks, we must confront the general nonconvex case. Even convex optimization is not without complication.

Ill-Conditioning

The most prominent challenge is ill-conditioning of the Hessian matrix H in optimization problem. According to the Taylor series expansion, cost function over the step −ϵg will add as below.

12ϵ2gTHg−ϵgTg

Seriously ill-conditioning H means the first term will add more than the second term, which will increase the cost function.To determine whether ill-conditioning is detrimental to training task, we can monitor the squared gradient gTg and the gTHg.

Though ill-conditioning is present in other settings besides neural network training, some of the techniques used to combat it in other contexts are less applicable to neural networks, like Newton's method.

Local Minima

Convex optimization problem can be reduced to find a local minimum which is guaranteed to be the global minimum. Nonconvex optimzization problem, such as neural network training, may have lots of local minima which is not the global minimum. As we will see, this is not a major problem for neural network training.

Neural network have many local minima because of the model identifiability which is from weight space symmetry and scales equivalent property.

Local minima is problematic if its cost function is much higher than the minimum one. Whether networks of practical interest have many local minima of high cost and whether optimization algorithms encounter them remian open questions. For sufficiently large neural network, most local minima have a low cost function value, and that it is not important to find a ture global minimum rather than to find a point with low but not minimal cost.

A test that can rule out local minima as the problem is plotting the norm of the gradient over time. If the norm doesn't shrink to insignificant size, the problem is neither local minima and any other kind of critical points.

Plateau, Saddle Points and Other Flat Regions

In high-dimensional nonconvex functions, local minima are rare compared to the saddle points because of Hessian matrix H rarely have all the eigenvalues positive.

An amazing property of many random functions is that the eigenvalues of the Hessian become more likely to be positive as we reach regions of low cost. Some researches show that real neural networks also have loss functions that contain very many high-cost saddle points.

The implications of the proliferation of saddle points are interesting. For first-order optimization, algorithms that only use gradient information, the situation is unclear. It seems gradient descent empirically escapes saddle points in many cases. Continuous-time gradient descent may be shown analytically to be repelled from, rather than attracted to nearby saddle point.

For Newton's method, it's designed to seek a point with zero gradient that makes it jump to saddle point. Second-order optimization is difficult to apply to deep neural network training.

There may also be wide, flat regions of constant cost function value, which pose major problems for all numerical optimization algorithms.

Cliffs and Exploding Gradients

Neural networks with many layers often have extremely steep regions resembling cliff, because of the multiplication of several large weights together. As using the gradient descent optimization algorithms, the gradient update step can move the parameters extremely far, usually jumping off the cliff.

The most serious consequences can be avoided by using gradient clipping heuristic algorithms.

Long-Term Dependencies

When neural networks go very deeply, the long-term dependencies problem arise. Typically , recurrent networks multiply the same weight repeatedly has the problem.

Suppose the weight matrix W=Vdiag(λ)V−1. In this simple case, it's straightforward to know as below.

Wt=(Vdiag(λ)V−1)t=Vdiag(λt)V−1

When t is large, the gradient is going to vanishing or exploding which depends on the value of λ. Likewise, only the principal eigenvector take effect as the x go through the neural network.

Recurrent networks use the same W at each time step, but MLP do not, so even very deep MLP can largely avoid vanishing and exploding gradient problem.

Inexact Gradients

Most optimization algorithms are designed with the assumption that we have exact gradient and Hessian matrix. In practice, we usually have a noisy or biased estimator of the these quantities.

In other cases, the objective function is intractable, so as the gradient and Hessian matrix. So, we actually only can approximate it.

Various neural network optimization algorithms are designed to account for imperfections in the gradient estimate. One can also tackle with the intractable case by choosing a surrogate cost function.

Poor Correspondence between Local and Global Structure

Many problems we discussed so far is about the properties of the cost function at a single point. It is possible to perform poorly without these problems, because the local improvement direction is not toward distant regions of much lower cost.

Much of the runtime of training is due to the length of the trajectory needed to arrive at the solution. Future research will need to develop further understanding of the factors that influence the length of the learning trajectory and better characterize the outcome.

Regardless of which of these problems are most significant, all of them might be avoided if there exists a region connected to the best solution by a path that local descent can follow, and if we are able to initialize learning within that well-behaved region.

Theoretical Limits of Optimization

Even several theoretical results show that there are limits on the performance of optimization algorithms we designed, these results have little bearing on training neural network in practice.

Basic Algorithms

The gradient descent algorithm is common used in objective function optimization, this may be accelerated significantly by using stochastic gradient descent of randomly selected minibatches.

Stochastic Gradient Descent

As discussed before, it is possible to obtain an unbiased estimate of the gradient by taking the average gradient on a minibatch of m examples drawn i.i.d from the data-generating distribution.

A crucial parameter for the SGD algorithm is the learning rate. In practice, it is necessary to gradually decrease the learning rate over time, we denote it as ϵk.

A sufficient condition to guarantee convergence of SGD is below.

∑ϵk=∞ and

∑ϵk2<∞

In practice, it's common to decay the learning rate in a linear manner until iteration τ.

ϵk=(1−α)ϵ0+αϵτ

Where α=kτ. As iteration excess τ, it is common to leave ϵ constant.

The learning rate may be chosen by trial and error, we can plot objective function over time to monitor the learning curves. This is more of an art than a science, and most guidance on this topic should be regarded with some skepticism.

When using linear schedule, usually τ may be set to the number of iterations required to make a few hundred passes through the training set. ϵτ shouble be set to roughly 1 percent the value of ϵ0. The problem is how to set ϵ0, if it's too large, the learning curve will show violent oscillation, if it's too small, learning proceeds slowly and may stuck with a high cost value. Typically, the optimal initial learning rate, in term of total training time and the final cost value, is higher than the learning rate that yields the best performance after the first 100 iterations or so. Therefore, it is usually best to monitor the first several iterations and use a learning rate that is higher than the best-performing learning rate at this time, but not so high that it causes severe instability.

The most important property of SGD is that time per update does not grow with the number of training examples. For a large enough dataset, SGD may converge to within some fixed tolerance of its final test set error before it has processed the entire training set.

Excess error J(θ)−minθJ(θ) is common used to study the convergence rate of an optimization algorithm. When SGD is applied to a convex problem, the excess error is O(1k) after k iterations, while in the strong convex case, it is O(1k). Some research argue that it therefore may not be worthwhile to pursue an optimization algorithm that converges faster than O(1k) for machine learning tasks -- faster convergence presumably corresponds to overfitting.

Momentum

The momentum method is designed to accelerate the learning process, especially in the face of high curvature, noisy gradient and small but constant gradient.

Momentum method introduces a velocity variable v to incorporate the historical gradient. The update rule is as below.

v←αv−ϵ1m∑ΔθJ(θ)

θ←θ+v

Previously, the size of the step was simply the norm of the gradient multiplied by the learning rate. Suppose we continues to observe the same gradient g, the size of each step is limit close to ϵ||g||1−α. The accelerated rate is 11−α, common values of α include 0.5, 0.9, 0.99 in practice.

Like the learning rate, α can also be adapted over time by increasing. Typically, adapting α is less important than shrinking learning rate over time.

Momentum method is just like adding a viscous drag to the descent process, which make it much faster to land on the minimum by avoiding moving excessive back and forth along the path. And velocity is a appropriate force which is not too weak to prevent the unjustified moving, or not too strong to stuck before reached the minimum.

Nesterov Momentum

Nesterov momentum is a variant of the momentum method, which evaluates the gradient after the current velocity is applied. Thus one can interpret Nesterov momentum method as attempting to add a correction factor to the standard method of momentum. The update rule is as below.

v←αv−ϵ1m∑ΔθJ(θ+αv)

θ←θ+v

In the context of convex batch gradient case, Nesterov momentum brings the rate of convergence of the excess error from O(1k) to O(1k2). Unfortunately, in the SGD case, Nesterov momentum doesn't improve the rate of convergence.

Parameters Initialization Strategies

Initial point is very important for training neural networks in terms of several aspects. It can determine whether the algorithm convergences at all, with some initial points being so unstable that the algorithm encounters numerical difficulties and fails altogether. It can affect the how quickly converges and whether it converges to a point with low or high cost. Also, it can affect the generalization error as well.

Modern initialization strategies are simple and heuristic, because neural network optimization is not yet well understood. Sometimes some initial points may be good for optimization, but detrimental for generalization, our understanding of how the initial point affects generalization is especially primitive.

Perhaps the only property known with complete certainty is that the initial point need to break symmetry between different units. If two units have the same function on the same inputs, deterministic learning algorithm applied to a deterministic obejective function will constantly update both in the same way. There are some methods to tackle with this, like Gram-Schmit orthogonalization on a initial weight matrix and be guaranteed that each unit would compute a very different function from each other unit.

Typically, we set the biases for each unit to heuristically chosen constants, and initialize only the weights randomly. Extra parameters, such as parameters encoding the conditional variance of a prediction, are usually set to heuristically chosen constants much like the biases are.

All the weights are commonly initialized to values drawn randomly from a Gaussian or uniform distribution. The choice of distribution do not seems to matter much, but the scale of the distribution matters.

Larger initial weights will yield a stronger symmetry-breaking effect, helping to avoid redundant units. Too large weights will lead to exploding values during forward propagation or back-propagation. Large weights may also result in extreme values that cause the activation function to saturate. These factors set the trade-off.

The optimization and regularization have different insights about parameters initialization. Optimization's perspective suggests we shoule hold a large enough initialized weights to propagate information and make the learning process fast. Otherwise, regularization's perspective commonly suggests a small initialized weights to achieve a good generalization, like the weight decay algorithm does.

Some heuristics are avaliable for choosing the initial scale of weights. One suggests each weight drawn from a uniform distribution U(−1m,1m), the other one suggests the distribution to U(−6n+m,6n+m), where m,n stands for inputs, outputs respectively. The latter is designed to compromise between activation variable and gradient variance with the assumption that no nonlinearities exist. Many strategies designed for the linear model perform well on nonlinear counterparts too.

Saxe recommend initializaing weights to random orthogonal matrices, with a carefully chosen scaling or gain factor g, Sussillo showed that setting the gain factor correctly is sufficent to train a deep network without orthogonal requirement. In the linear assumption, networks can avoid the vanishing and exploding gradients problem that arises when the same weight matrix is used at each step.

Unfortunately, these optimal criteria for initial weights often do not lead to optimal performance with some possible reasons. In practice, we usually need to treat the scale of initial weights as a hyperparameter whose optimal value lies roughly around the theoretical predictions.

One drawback to scaling all initial weights in the same deviation is that every initial weight become extremely small relative to entire large network. Martens introduces sparse initialization, in which some weights are chosen to 0. This method enhances the diversification of weights but cause some other problems. We need prior knowledge to determine which weights set to 0. Also, it's not adaptive for such as maxout units.

If we have enough computational resources, it is usually good to search a good scale for initial weight by monitoring the range or deviation of activations or gradients on a single minibatch of data.

Setting biases needs to be coordinated with weight initialization schemes. Setting biases to 0 is compatible with most weight initialization schemes, only few situations where we may set to nonzero values.

  • If a bias is for an output unit, it is often benificial to initialize the bias to obtain the right marginal statistics of the output.
  • Sometimes we may want to choose the bias to avoid too much saturation at initialization.
  • Sometimes a unit controls whether other unit can participate the forward propagation, it has better set the bias to 1 to open the gate initially.

Another common type of parameter is the precision or variance, such as the variance in the linear regression. It's usually set to 1 safely, but also can be set to marginal variance of the output in the training set if weights near 0 and biases near marginal mean.

Besides these simple constant or random methods of initializing method, we can also using machine learning to find out a good initial parameters. Some initialization strategies may yield faster convergence and better generalization because they encode information about the distribution in the initial parameters of the model. Others apparently perform well primarily because they set the parameters to have the right scale.

Algorithms with Adaptive Learning Rates

Learning rate is the most difficult hyperparameter which significantly affects the neural network training and generalization. If we believe the cost sensitivity depends on axis aligned, it's reasonable to adapt the learning rate of different parameters throughout the course of learning.

AdaGrad

AdaGrad algorithm adapts the learning rates of parameters by scaling them inversely proportional to the square root of the sum of historical squared values of gradient.

In convex optimization, AdaGrad can enjoy some good properties. In neural network training, AdaGrad often result in a premature and excessive decrease in the effective learning rate. AdaGrad performs well for some but not all deep neural networks.

RMSProp

RMSProp algorithm modifies AdaGrad by changing cumulation of all historical gradient into exponential moving average which discard the extremely past gradient information.

RMSProp avoid the premature and excessive decrease in the learning rate, which accelerate the convergence. It is currently one of the go-to optimization methods being employed routinely by deep learning practitioners.

Adam

Adam is another adaptive learning rate algorithm which can be seen as a combination of RMSProp and momentum with a few distinctions. First, momentum is incorporated with gradient directly. Second, bias correction is involved.

Adam is regarded as being fairly robust to the choice of hyperparameters, though the learning rate sometimes need to be changed from the suggested default.

Choosing the Right Optimization Algorithm

There is no consensus on the question which algorithm shoule be chosen. Currently, the most popular optimization algorithms actively in use include SGD, SDG with momentum, RMSProp, RMSProp with momentum, Adam. The choice of which algorithm to use, at this point, seems to deep on the familiarity with the algorithm.

Approximate Second-Order Methods

Second-Order method incorporates the Hessian matrix information into the learning process. Newton's method is typically second-order method.

Newton's Method

Newton's method is an optimization algorithm based on second-order Taylor series expansion to approximate cost function J(θ) near certain point θ0.

J(θ)=J(θ0)+(θ−θ0)g(θ0)+12(θ−θ0)TH(θ−θ0)

If we solve the critical point of this function, we obtain the update rule of Newton's method as below.

θ∗=θ−H−1g

Thus for a locally quadratic function, method achieve the minimum at one jump. If it is convex function, this update can be iterated, yielding the training algorithm associated with Newton's method.

Drawback are obvious, which includes the ill-condition Hessian matrix and computational burden imposed by calculation of Hessian matrix, finally lead to the rare use in deep learning.

Conjugate Gradients

Conjugate gradients is a method to avoid calculating the inverse of Hessian matrix by iteratively descending conjugate directions.

Let the previous search direction be dt−1, when reached the minimum, the new search direction is orthogonal to last one as below.

ΔθJ(θ)dt−1=0

Steepest method shows the path with zig-zag pattern, which will undo the progress of the previous search direction. Conjugate gradients seeks to address this problem.

In the conjugate gradients method, we set the new search direction conjugate to the previous one.

dt=ΔθJ(θ)+βtdt−1

where βt represent the magnitude controls how much previous direction involves. And if dtTHdt−1=0, we call dt and dt−1 are conjugate.

Two popular methods for computing β are as below.

  • Fletcher-Reeves:

    βt=ΔθJ(θt)TΔθJ(θt)ΔθJ(θt−1)TΔθJ(θt−1)

  • Polak-Ribiere:

    βt=(ΔθJ(θt)−ΔθJ(θt−1))TΔθJ(θt)ΔθJ(θt−1)TΔθJ(θt−1)

For a quadratic surface, the conjugate directions ensure that the gradient along the previous direction does not increase in magnitude. We therefore stay at the minimum along the previous direction. It requires at most k searches to achieve minimum.

Deep learning network is far from quadratic objective function. However, with some modifications, nonlinear conjugate gradients is still applicable in deep learning network training.

BFGS

The Broyden-Fletcher-Goldfarb-Shanno (BFGS) algorithm attempts to bring the advantage of Newton's method without the computational burden.

The approach is to approximate the inverse with a matrix Mt that is iteratively refined by low-rank updates to become a better approximation of H−1. The search direction ρt is Mtgt for determining the size of step ϵ∗. The final update is given as below.

θt=θt−1+ϵ∗ρt

BFGS incorporate the second-order information, and the success is not heavy dependent on the line search finding a point which is close minimum. However, BFGS need to store the matrix approximator Mt which requires O(n2) memory. It makes BFGS impractical to deep learning network with millions of parameters.

Limited Memory BFGS algorithm can significantly reduce the memory costs by avoiding storing the complete inverse Hessian approximation M via begining with the assumption that Mt−1 is identity matrix. This procedure remains well behaved when the minimum of the line search is reached only approximately.

Optimization Strategies and Meta-Algorithms

Many optimization strategies are not single algorithm, but can be incorporated into many different algorithms.

Batch Normalization

Batch Nomalization is a adaptive reparametrization motivated by the neural network training difficulty.

In practice, we update all the layers parameters simultaneously, the change of objective function is not always accurately approximated base on gradients, especially when we have large weights. Batch normalization put a standard transformation on each activation, which erase the bias and variance influence of previous layers.

To maintain the expressive power, it is usually add some scale and bias to the normalization units.

For most nonlinear neural network ϕ(XW+b), Ioffe and Szegedy recommend to apply the batch normalization to XW+b. Because the output of a nonlinear activation are thus more non-Gaussian and less amenable to standardization by linear operations.

Coordinate Descent

In some cases, it may be possible to solve an optimization problem quickly by breaking it into separate pieces. Coordinate descent algorithm minimizing with respect to a subset of the variables simultaneously, then minimizing another subset, finally arrive at a minimum.

When optimization problem can separate into groups that play relatively role, coordinate descent makes the most sense. Otherwise, when one variable strongly influences the optimal value of another variable, coordinate descent performs not well.

Polyak Averaging

Polyak averaging consists of averaging several points in the trajectory through parameter space visited by an optimization algorithm. The basic idea is that optimization algorithm may leap back and forth across a valley, the average will be close to the bottom of the valley.

In convex problems, the arithmetic average is commonly used. For nonconvex problems, exponentially decaying running average is commonly used.

Supervised Pretraining

Sometimes, directly training a complex model is too difficult, it is very useful to break the problem into many components, then solve the separate problem efficiently, finally combining each optimal component together. It is called greedy algorithms. For supervised learning, we call it greedy supervised Pretraining.

In the original version, each stage consists of a supervised learning training task involving only a subset of the layers in the final neural network.

An approach related to supervised pretraining extends the idea to the context of transfer learning. Pretrain a deep convolutional net with eight layers of weights on a set of tasks (a subset of 1000 ImageNet object categories) and initialize a same-size network with the first k layers of the first net. All the layers are then jointly trained to perform a different set of tasks.

Another approach is the FitNets, which begins by training a network that has low enough depth and great enough width to be easy to train. This network becomes a teacher for a second network, designated the student. Other optimization techniques or changes in the architecture may also solve the problem.

Designing Models to Aid Optimization

In practice, it is more important to choose a model family that is easy to optimize than to use a powerful optimization algorithm. Specifically, modern neural networks reflect a design choice to use linear transformations between layers and activation functions that are differentntiable almost everywhere, with significant slope in large portions of their domain.

Other model design strategies can help to make optimization easier, like linear paths or skip connection between layers reduce the length of the shortest path from the lower layer's parameters to the output, and thus mitigate the vanishing gradient problem.

Continuation Methods and Curriculum Learning

Continuation methods are a family of strategies that can make optimization easier by choosing initial points to ensure that local optimization spends most of its time in well-behaved regions of space. The idea behind continuation methods is to construct a series of objective functions over the same parameters. The solution to one is a good initial point to the next.

Curriculum learning can be interpreted as a continuation method, the idea is planning a learning process to begin by learning simple concepts and progress to learning more complex concepts that depend on these simpler concepts. This basic strategy was previously known to accelerate progress in animal training and in machine learning.
