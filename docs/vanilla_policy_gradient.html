Vanilla Policy Gradient Algorithm

Simple policy gradient algorithm introduces the basic idea of policy gradient improving the policy directly via interacting with environment . The sample average is used to estimate the gradient, how to make this estimator close to the true gradient is the key issue.

Bias vs Variance

So what means close for estimator? In Statistics, the Mean Square Error(MSE) is widespread used for measuring the error between estimator and parameter.

Generally, if statistic X^ is the estimator of parameter μ.

MSE(X^)=E[(X^−μ)2]=∫(X^−μ)2dp

Actually, you can use any distance function replaces the square measurement, like absolute distance, etc. Square distance has some mathmatical convenients, like calculating gradient, etc. We will use mean square error in this thesis.

MSE Error Natural Decompose into Bias and Variance.
MSE(X^)=E[(X^−μ)2=E[(X^−E(X^)+E(X^)−μ)2=E[(X^−E[X^])2]+E[(E[X^]−μ)2]+2E[(X^−E[X^])(E[X^]−μ)]=E[(X^−E[X^])2]+E[(E[X^]−μ)2]=Variance(X^)+Bias(X^)

Then, the first part (variance) measures the estimator's stability, the second part (bias) measures the natural difference. According to big number law, the first part will close to zero in probability as the sample number increase to the infinite, the second part will never change. That's why we care much more about bias in theory, we call it unbiased estimator with zero bias.

In practice, making experiments to collect data have a cost, sometimes heavy, bias-variance trade-off is a key issue.

Find a better estimator of policy gradient

As derivation in simple policy gradient document, 1Nτ∑iG(τi)∇θ∑klogπθ(ai,k|si,k) is a unbiased estimator of policy gradient, we try to improve this estimator in way of lower variance.

Reward to Go

We just look into G(τ)∇θ∑klogπθ(ak|sk)∼Dθ , repeat sampling decrease the variance in a solid way. Make it clear: (a, s, r) is random variables.

G(τ)=∑krk multiply every logπθ(ak|sk) is actually importing some unnecessary variance, cause the reward before k has no relevant to ak. Normally, reward is either positive or negative, it's easy to prove ∇θ∑klogπθ(ak|sk)(∑krj) has a smaller variance than G(τ)∇θ∑klogπθ(ak|sk). We call the new estimator reward-to-go policy gradient estimator[1].

Unbiased Estimator

According to the MSE Natural Decoposition, if the estimator is unbiased with lower variance must hanve a smaller MSE than simply policy gradient estimator. Unbiased property proof is obvious.

First, we will show this.

E[∇θlogπθ(ak|sk)(∑k−1rj)]=E[∇θlogπθ(ak|sk)]E[(∑k−1rj)]=∫πθ(ak|sk)∇θlogπθ(ak|sk)dpE[(∑k−1rj)]=∫∇θπθ(ak|sk)dpE[(∑k−1rj)]=∇θ∫πθ(ak|sk)dpE[(∑k−1rj)]=0

Then,

E[∇θ∑klogπθ(ak|sk)(∑krj)]=E[G(τ)∇θ∑klogπθ(ak|sk)]−E[∇θlogπθ(ak|sk)(∑k−1rj)]=∇θE[G(τ)]

Intution of variance reduction

Assume ∇θlogπθ(ak|sk) is the direction increasing the probability of ak at sk, G(τ) or ∑krj is the weight decide how much to go in that direction. If we use G(τ) what means no differences in the episode, comparison in different episodes figures out the final direction, so it's episode efficient. For reward-to-go, the weights changes in episode, so it's step efficient.

Another intuition is purifying, we filter some noise information, like the past rewards. The more the algorithm filter, the less variance we get.

General Form of the Policy Gradient

Follow the above idea, continue to purify the weight information. let the Ψt denote the weight [2], the general form of vanilla policy gradient is below.

∇θE[G(τ)]=E[∑k∇θlogπθ(ak|sk)Ψt]

where Ψt could be any transformation which satisfies the above equation.

like

Ψt=G(τ)

or

Ψt=∑krj

or

Ψt=∑krj−b(st)

The last formula b(st) represents the noise information we should purify. What's important we should know, is Ψt is not neccessary built on the reward random variable. The value function is also valid[2] [3], this is obviously with conditional probability.

like action-value function

Ψt=Qπ(st,at)

or advantage function

Ψt=Aπ(st,at)=Qπ(st,at)−Vπ(st)

or TD residual (another advantage)

Ψt=Vπ(st+1)+rt−Vπ(st)

or any formula represents the action value at st.

Consider the actual value function known, the last three weight schemes must have a lower variance than the above three. Cause the function is not a random variable, which means zero variance. But the truth is mostly we don't know the world dynamic at all, we need to figure out the value function by agent-self.

Sutton proves approximation function fw(s,a) need to have a orthogonal gradient with logπθ(s,a), in order to make the new estimator (replace value function with approximation function) unbiased. The condition is too strong to follow, mostly approximate of value function will impose the bias error.

So, how to design the weight scheme shows the consideration about bias-variance trade-off.

From the purifying perspective, advantage will be the best form which delete the unnecessary information most. Think about it, the weight be positive for good action, negative for bad action, which lead to rise or fall the action probability via change θ.

Generalized Advantage Estimation

Schulman gives a generalized advantage estimation formula, which i guess from idea TD(λ)[2]. The weight schem has a parameter λ to control the bias-variance trade-off. As λ increased, bias gets smaller and variance gets larger. Additional, the approach is to estimate the state value function.

Let one-step advantage (TD Residual )

δt=Vπ(st+1)+rt−Vπ(st)

Let more steps advantage

A^t(n)=δt+δt+1+...+δt+n−1=Vπ(st+1)+rt−Vπ(st)+Vπ(st+2)+rt+1−Vπ(st+1)+...+Vπ(st+n)+rt+n−1−Vπ(st+n−1)=∑tt+n−1rk+Vπ(st+n)−Vπ(st)A^t(∞)=∑trk−Vπ(st)

A^(1) only use one step reward, rest estimated by approximation function, which means big bias, small variance.

A^(∞) only use reward-to-go, has no bias, but larger variance.

Generalized advantage estimation uses exponentially-weighted to deal with this all advantage estimators.

A^tGAE(λ)=(1−λ)∑1∞λk−1A^t(k)

After expansion, we can find below, detail could be found in [2].

A^t(λ)=∑0∞λkδt+kA^tGAE(0)=δt,  A^tGAE(1)=A^(∞)

Vanilla Policy Gradient Algorithm

With generalized advantage estimation, VPG estimate the policy gradient using state value approximation.

Pseudocode

Initilize the policy parameters θ0 and the state value function parameters w0

for i = 0, 1, 2, .... do

Simulate current policy πθ for k episodes, collect the trajectories data {τj,j=1,2,..k}

for j = 1,2,3...k do

for t = 0, 1, ..., termial do

calculate δt=fw(st+1)+rt−fw(st)

end for t

for t = 0, 1, ..., termial do

calculate A^tGAE(λ)

calculate reward-to-go Rj,t=∑m=trj,m store with state sj,t in pairs set {(R,S)}

end for t

          calculate eposide gradient estimator $g_j ={\nabla}_{\theta}{\sum}log\pi_{\theta}(a_t|s_t)\hat{A}^{GAE(\lambda)}_t$

end for j

calculate the final gradient estimator gi=∑gj/k

update θ via gradient ascent algorithm like RMSProp

repeat many times

calculate the w gradient gw of 1|(R,S)|∑r,s∈(R,S)(r−fw(s))2

update w via graient descent algorithm like RMSProp

end for i
